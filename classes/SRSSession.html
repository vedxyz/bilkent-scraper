Static withAutomatedVerification

  • withAutomatedVerification(id: string, password: string, email: string, emailPassword: string, boxname?: string): Promise<SRSSession>
    • Defined in srs/session.ts:76

    Allows you to authenticate with SRS fully automatically, by also providing email credentials to retrieve the verification code with (get2FACode).

    In order to use this authentication flow, some prior configuration is required. The SRS user must have set up a mail filter to direct all verification codes to a specific mailbox/folder. See package documentation for more details.

    see

    withManualVerification for an alternative

    Parameters

    • id: string

      Bilkent student ID

    • password: string

      Bilkent SRS password

    • email: string

      Bilkent email address (where authentication codes are sent)

    • emailPassword: string

      Password for the email address

    • boxname: string = "STARS Auth"

      The mailbox/folder containing SRS authentication codes (defaults to "STARS Auth")

    Returns Promise<SRSSession>

    An authenticated SRS session

Static withManualVerification

  • withManualVerification(id: string, password: string): Promise<{ reference: string; verify: any }>
    • Defined in srs/session.ts:46

    Allows you to authenticate with SRS the typical way, by manually providing a verification code received as SMS/email.

    The reference code for the verification code is returned along with a verify function to submit the corresponding verification code.

    see

    withAutomatedVerification for an alternative

    • password: string

      Bilkent SRS password
